# Доступ к Kibana/Elastic с телефона по Wi‑Fi — чеклист

## 1. Один ли у вас Wi‑Fi

- Ноутбук и телефон должны быть в **одной сети**: один и тот же SSID (имя точки доступа), без гостевой сети.
- У ноутбука IP `192.168.8.54` — значит сеть скорее всего `192.168.8.0/24`. У телефона в настройках Wi‑Fi должен быть IP вида `192.168.8.xxx` (тот же диапазон).

## 2. Изоляция клиентов (AP Isolation) в роутере

Частая причина: в роутере включена **изоляция клиентов Wi‑Fi** (Client/AP Isolation). Тогда устройства в Wi‑Fi не видят друг друга, только интернет.

Что сделать:
- Зайти в веб-интерфейс роутера (часто `http://192.168.8.1` или `http://192.168.8.254`).
- Найти раздел вроде: **Wi‑Fi** / **Беспроводная сеть** / **Дополнительно**.
- Искать опции: **«Изоляция клиентов»**, **«AP Isolation»**, **«Client Isolation»**, **«Изоляция беспроводных клиентов»**.
- **Отключить** эту опцию и сохранить настройки.

(Если такой опции нет — можно пропустить и проверить п.3.)

## 3. Проверка связи телефон ↔ ноутбук

- На телефоне: установить любое приложение для проверки сети (например «Ping», «Network Utilities») и **пинговать** `192.168.8.54`.
  - Пинг идёт — сеть между телефоном и ноутбуком есть, проблема скорее в портах/сертификате.
  - Пинга нет — между устройствами трафик не ходит (часто изоляция в роутере или разные сети).

## 4. На ноутбуке уже настроено

- Portproxy: порты 5601, 9200, 8220, 8200 перенаправляются с `0.0.0.0` на `127.0.0.1`.
- Брандмауэр: правила «Elastic Kibana», «Elasticsearch», «Fleet Server», «APM Server» — разрешают входящий трафик (Profile: Any).

## 5. Если изоляцию отключили — попробовать снова

- С телефона в браузере: `https://192.168.8.54:5601`
- При предупреждении о сертификате — принять/продолжить (самоподписанный сертификат).

## 6. Точка доступа с iPhone

Если ноутбук подключён к **точке доступа iPhone**:
- У ноутбука в этой сети будет IP вида **172.20.10.x** (сейчас у тебя **172.20.10.5**).
- С **iPhone** (того, кто раздаёт точку) открывай Kibana по адресу ноутбука: **https://172.20.10.5:5601**
- Не путай: открывать нужно на **iPhone**, а в адресе — **IP ноутбука** (172.20.10.5), не телефона.

Если страница не открывается — см. п. 7 (брандмауэр для подсети точки).

## 7. Правило брандмауэра для подсети точки доступа

Иногда общие правила не срабатывают для новой сети. Запусти **PowerShell от имени администратора** и выполни:

```powershell
New-NetFirewallRule -DisplayName "Elastic from Hotspot" -Direction Inbound -LocalPort 5601,9200,8220,8200 -Protocol TCP -Action Allow -RemoteAddress 172.20.10.0/24
```

После этого с iPhone снова открой **https://172.20.10.5:5601**

## 8. Альтернатива: мобильная точка доступа с ноутбука

Если роутер менять нельзя или изоляцию отключить нельзя:
- Раздать интернет с **ноутбука** через «Мобильный хот-спот» (Windows: Параметры → Сеть и интернет → Мобильный хот-спот).
- Подключить **телефон** к этой точке доступа.
- Тогда телефон и ноутбук в одной маленькой сети без изоляции; пробовать снова `https://192.168.8.54:5601` (IP ноутбука в хот-споте может быть другой, например `192.168.137.1` — тогда с телефона использовать этот IP).
